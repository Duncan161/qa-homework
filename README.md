## Описание
Учебный проект для демонстрации автоматизации контроля качества кода с помощью GitHub Actions.

## Реализованные этапы автоматизации
1. **Статический анализ (Linting):** Используется `pylint`. Проверка стиля кода, длины строк (макс 100), наличия неиспользуемых импортов. Порог оценки: 7.0.
2. **Автоматическое тестирование:** Используется `pytest`. Написано 5 unit-тестов для функций калькулятора.
3. **Проверка покрытия кода (Quality Gate):** Используется `pytest-cov`. Пайплайн падает, если покрытие кода менее 70%.
4. **Артефакты:** Отчет о покрытии сохраняется как артефакт сборки.

## Вывод
Автоматизация позволила:
- Быстро находить ошибки стиля и логики до мержа кода.
- Гарантировать, что новый код не снижает общее покрытие тестами ниже установленного порога.
- Избежать ручного запуска проверок перед каждым коммитом.
